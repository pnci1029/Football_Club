# 공통 개발 가이드

## 프로젝트 개요
- **프로젝트명**: Football Club
- **타입**: 조기 축구 동호회 개별 커스텀 소개 페이지
- **아키텍처**: Kotlin(Backend) + React(Frontend)
- **주요 사용자**: 조기 축구 클럽 회원 및 관리자
- **디자인 레퍼런스**: [대구FC](https://www.daegufc.co.kr/main/)

## 비즈니스 요구사항

### 핵심 기능 (우선순위별)
1. **최고 우선순위**: 축구 회원 정보 및 사진 관리/노출
2. **중간 우선순위**: 구단별 구장 정보 관리
3. **낮은 우선순위**: 대결 신청 및 예약 시스템

### 서브도메인 구조
- 각 구단별로 서브도메인을 통해 정보 분리
- 예시: `team-a.footballclub.com`, `team-b.footballclub.com`
- 관리자 페이지: `admin.footballclub.com`

#### 관리자 서브도메인
- **형식**: `admin.~host.co.kr~` 
- **목적**: 전체 시스템 관리, 다중 팀 관리
- **접근 권한**: 시스템 관리자 전용
- **기능**: 팀 생성/수정/삭제, 서브도메인 관리, 전체 통계

#### 개별 동호회 서브도메인  
- **형식**: `{club-name}.~host~` (예: `kim.~host~`, `park.~host~`)
- **목적**: 각 동호회별 독립적인 웹사이트 운영
- **관리 방식**: 
  - 각 동호회별로 독립된 데이터 및 관리 영역
  - 서브도메인을 통한 멀티테넌트 아키텍처
  - 동호회별 맞춤형 콘텐츠 및 설정

### 주요 데이터
- **회원 정보**: 이름, 포지션, 프로필 사진, 개인 통계
- **구장 정보**: 위치, 시설, 예약 가능 시간, 요금
- **대결 기록**: 경기 일정, 결과, 통계

## 개발 원칙

### 개발 도구 및 환경
- **패키지 매니저**: Yarn 사용 (npm 대신)
- **프론트엔드**: React + TypeScript + Tailwind CSS
- **백엔드**: Kotlin + Spring Boot
- **빌드 도구**: Gradle (Kotlin DSL)

### 코드 품질
- 코드 리뷰 필수
- 테스트 커버리지 80% 이상 유지
- 린터 및 포맷터 규칙 준수
- 명확한 변수명과 함수명 사용

### 보안
- 환경변수로 민감정보 관리
- API 키와 비밀번호 절대 하드코딩 금지
- HTTPS 통신 필수
- 입력값 검증 철저히 수행

### 성능
- 모바일 최적화 우선 고려
- 이미지 최적화 (WebP, 적절한 해상도)
- 번들 사이즈 최소화
- 로딩 시간 3초 이내 목표

## Git 컨벤션

### 브랜치 전략
- `main`: 프로덕션 브랜치
- `develop`: 개발 통합 브랜치
- `feature/기능명`: 기능 개발 브랜치
- `hotfix/이슈명`: 긴급 수정 브랜치

### 커밋 메시지
```
type(scope): description

[optional body]
[optional footer]
```

**타입:**
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `style`: 코드 스타일 변경
- `refactor`: 리팩토링
- `test`: 테스트 추가/수정
- `chore`: 빌드 프로세스, 도구 설정

## API 설계 원칙

### RESTful API
- 명사형 엔드포인트 사용
- HTTP 메서드 적절히 활용
- 상태 코드 정확히 반환

## 전체 프로젝트 개발 워크플로우

### Phase 1: 기반 설정 (1-2주)
1. **프로젝트 구조 설정**
   - 백엔드: Kotlin + Spring Boot 프로젝트 초기화
   - 프론트엔드: React + TypeScript + Tailwind 설정
   - Git 리포지토리 및 브랜치 전략 수립

2. **개발 환경 구축**
   - 로컬 개발 환경 설정
   - 데이터베이스 초기 스키마 설계
   - API 명세서 초안 작성

### Phase 2: 핵심 기능 개발 (3-4주)
1. **백엔드 핵심 API**
   - 팀/선수 엔티티 및 API 구현
   - 서브도메인 기반 멀티테넌시
   - 파일 업로드 (선수 사진) 기능

2. **프론트엔드 핵심 UI**
   - 네비게이션 및 레이아웃 구현
   - 선수 관리 페이지 (최우선)
   - API 연동 및 상태 관리

### Phase 3: 확장 기능 개발 (2-3주)
1. **구장 정보 시스템**
   - 백엔드: 구장 관리 API
   - 프론트엔드: 구장 정보 페이지

2. **관리자 기능**
   - 관리자 인증 시스템
   - 관리자 대시보드

### Phase 4: 마무리 및 배포 (1-2주)
1. **성능 최적화 및 테스트**
2. **배포 및 도메인 설정**
3. **사용자 피드백 수집 및 개선**
